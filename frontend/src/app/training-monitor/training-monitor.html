Neural Network Architecture

The architecture is the model's "brain shape". We use PPO (Proximal Policy Optimisation) — a reinforcement learning algorithm that learns by trial and error, adjusting its strategy gradually so it doesn't overshoot. PPO is wrapped around an LSTM (Long Short-Term Memory) network, which gives the model a sense of recent history — essential for betting where the market moves second by second.

Select which architectures should compete in the population. Agents are randomly assigned one of the selected architectures. If you select multiple, agents with different architectures will breed together — this explores more strategies but makes comparison harder.

Genetic Algorithm

We don't train one model — we train a population of models with randomly-sampled hyperparameters (learning rate, network size, etc). After each generation, the best are kept and "bred" to create the next generation, similar to natural selection. Over time, good hyperparameter combinations win out.

Population Size

Population size is how many agents train simultaneously per generation. Each has different random hyperparameters, so they explore different strategies. Larger populations find better solutions but multiply training time linearly — double the population, double the time.

For getting started or testing changes, 5-10 agents is enough. For serious runs, 30-50 agents is the sweet spot. Above 50 shows diminishing returns.

} @case (6) {

Training Parameters

Generations = how many evolutionary cycles to run. Each generation trains every agent, evaluates them, breeds the best, and repeats. More generations = more refinement but more time.

Epochs per day = how many times each agent processes each training day. More epochs means deeper learning from the same data, but too many cause "overfitting" — memorising specific races instead of learning general patterns. Start with 3 epochs.
