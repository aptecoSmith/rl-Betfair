# Frontmatter contract

Every Wiki note begins with YAML frontmatter delimited by `---`. To keep the core tool
**stdlib-only**, frontmatter uses a constrained YAML subset (see "Parser subset" below) ‚Äî
still valid YAML, still Obsidian-readable.

## Wiki note frontmatter

```yaml
---
id: 01J9Z3K8Q4R2N7B5C6D8E0F1G2  # required (v3) ‚Äî permanent ULID; added by `stamp-ids`, never hand-edit
type: concept            # required ‚Äî one of: topic, concept, entity, project, synthesis, log, query
subtype: person          # entity only ‚Äî person | group | org | product | tool | place
cloud: shared            # required ‚Äî shared | personal
status: seed             # required ‚Äî seed | draft | stable
created: 2026-05-30      # required ‚Äî ISO-8601 date
updated: 2026-05-30      # required ‚Äî ISO-8601 date
tags: [research]         # optional ‚Äî controlled context tags (see tag-vocabulary.md)
sources: [src-a1b2c3]    # source IDs into Schema/sources.jsonl (see below)
aliases: [PPO, ppo]      # optional ‚Äî alternative names for search
---
```

### Granular object IDs (v3, Phase A)
- `id` is a **ULID** ‚Äî a 26-char Crockford-base32 string (a 48-bit ms timestamp + 80 random bits),
  generated stdlib-only. It is the note's **permanent identity, independent of filename or path**:
  rename or move the file freely and links by id still resolve.
- IDs live **in the files** (the source of truth). Run `wiki_tool.py stamp-ids` to add a stable `id:`
  to any note lacking one ‚Äî it is **idempotent** (a note that already has a valid id is untouched).
- The derived **filename‚Üîid alias index** (regenerated by `build` into `.runtime/alias-index.json`,
  gitignored) lets legacy `[[wikilink]]` text keep resolving; it is never authoritative.

### Linking by id vs by filename
- **Canonical, rename-safe:** `[[id:01J9Z3K8Q4R2N7B5C6D8E0F1G2]]` (optionally with display text,
  `[[id:01J9‚Ä¶|Readable name]]`). The resolver **prefers id**.
- **Legacy/back-compat:** `[[some-note]]` resolves by filename stem via the alias index, and **warns on
  ambiguity** (a stem shared by two notes) ‚Äî disambiguate with the id form.
- `wiki_tool.py stamp-ids --relink` rewrites resolvable `[[stem]]` links into `[[id:‚Ä¶|stem]]`.

### Typed links (v3, Phase C)
Frontmatter may carry **typed edges** giving links meaning:
`links: [{to: <id>, type: <type>}]`. The `type` must be in the controlled vocabulary
(`Schema/link-types.md`: epistemic / structural / entity families) ‚Äî an **unknown type fails
`validate`**. Bare body `[[ ]]` links stay untyped `see-also`. Typed edges are **data, not LLM
inference**: `wiki_tool.py relations --from <note>` answers "who is X's sibling/parent/‚Ä¶" from the
graph (the inverse is derived, so you record an edge once). `supersedes`/`contradicts` edges feed the
audit (`relations --stale` / `--contradictions`).

### Claims & provenance (v3, Phase B)
A note is a **composition of claims**. Each claim ‚Äî a statement plus *where it came from* ‚Äî lives in a
**per-note sidecar** `<note>.claims.jsonl` (one JSON object per line, committed; the source of truth).
Sidecars were chosen over in-note `^block` anchors for clean diffs and trivial projection (see
`DECISIONS.md`). One claim:

```jsonc
{"id":"01J9‚Ä¶","text":"BVA tests the edges of equivalence partitions.","source_id":"src-a1b2c3",
 "locator":{"page":42,"quote":"edges of equivalence partitions","char_range":[1180,1212]},
 "asserted_by":"model:claude-opus-4-8","confidence":0.95,"created":"2026-06-04"}
```
- `locator` records **where in the source**: any of `page/pages/section/paragraph/slide/sheet/char_range`
  plus the lifted **`quote`** (the verifiable anchor ‚Äî survives even if the source later goes missing).
- `asserted_by` is `model:<name>` (LLM) or `human`; **`verified_by`** is `none` (default) ‚Üí `human` or
  `cross-source` (Phase F agency marking ‚Äî see `Schema/agency.md`). A model-asserted, unverified claim
  is the hallucination-risk flag. Promote with `wiki_tool.py verify`; nothing auto-verifies.
- `claims-lint` (run inside `finalize-ingest`) checks every claim: structure, that `source_id` is
  registered, and that the locator **resolves to a real span** in the source ‚Äî a quote not found in the
  source **fails** (the fabrication guard). Add claims with `wiki_tool.py claim-add`, which grounds the
  quote for you and refuses ungrounded provenance.
- A trailing ` ^blockid` on a body line remains **reserved** for future in-note block addressing; do
  not author `^` anchors by hand.

### Rules (enforced by `validate`)
- `id` present (warn if missing ‚Üí run `stamp-ids`); a present id is a valid, unique ULID.
- `type` present and in the allowed set; exactly one type per note.
- `subtype` only on `entity` notes, and from the entity-subtype set.
- `cloud` is `shared` or `personal`; must match the folder the note lives in.
- `status` in `seed|draft|stable`. For `draft`/`stable`, `sources` must be non-empty (every claim is sourced).
- `created`/`updated` are ISO-8601 dates.
- `tags` (if present) are all in the controlled context-tag vocabulary.
- Every id in `sources` exists in `Schema/sources.jsonl`.
- A `personal` note may link to `shared`; a `shared` note must NOT reference `personal` (cloud direction).

### Why source IDs, not inline source detail
The per-source detail (title, location, **per-machine presence**, hash) is the registry's job and
would be duplicated across every citing note. So the note frontmatter holds **IDs**; `build`
renders a human-readable `## Sources` block into the note body from the registry, so the location
and `present/missing` status still show up where you read the note. `source-check` keeps presence current.

## Source registry ‚Äî `Schema/sources.jsonl`

One JSON object per line:

```jsonc
{"id":"src-a1b2c3","title":"ISTQB CTFL Syllabus v4.0","content_type":"pdf",
 "sha256":"‚Ä¶","first_ingested":"2026-05-30",
 "locations":{"WORK-PC":{"path":"D:/docs/istqb/ctfl-v4.pdf","present":true,"last_seen":"2026-05-30"},
              "HOME-PC":{"present":false}}}
```
- `id` ‚Äî stable `src-` + short hash; referenced by notes.
- `locations` ‚Äî keyed on machine-id (from `.wiki-local.json`); `source-check` updates `present`/`last_seen`.
- A source absent on every machine is **dangling** ‚Äî allowed, but notes citing it are flagged `source_status: missing` (the quote lifted into the note keeps the citation meaningful).

## Source note (dropped files only)

Files dropped into an inbox get a lightweight companion note while pending; once compiled they
become registry entries and the inbox file moves to `processed/`. Reference-not-copy means external
sources never get a copied body ‚Äî only a registry record.

## Parser subset (what the stdlib parser accepts)
- `---` fences top of file.
- `key: scalar` (string, number, `true`/`false`, ISO date). Quotes optional; `"..."` honored.
- Inline lists: `key: [a, b, c]`.
- Block lists of scalars:
  ```yaml
  sources:
    - src-a1b2c3
    - src-d4e5f6
  ```
- Lines starting `#` are comments. No nested mappings (by design).
